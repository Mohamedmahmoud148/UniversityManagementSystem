# Complaint System

> **Last refreshed:** 2026-05-31

---

## 1. Overview

The Complaint System allows students to formally submit concerns about doctors, grades, exams, departments, or technical issues. Admin receives AI-generated intelligence reports (daily, weekly, monthly).

---

## 2. Complaint Entity

```csharp
Complaint {
    StudentId     // FK to Student
    Title
    Message       // max 2000 characters
    TargetType    // Doctor | Department | Admin | Technical | Subject
    TargetId?     // FK to the target entity
    Status        // Pending | UnderReview | Resolved | Dismissed
    Priority      // Normal | High | Critical
    CreatedAt
}
```

---

## 3. API Endpoints

| Method | Endpoint | Roles | Description |
|--------|----------|-------|-------------|
| POST | `/api/complaints` | Student | Submit complaint |
| GET | `/api/complaints` | Admin | All complaints (paginated, filterable) |
| GET | `/api/complaints/{id}` | Auth | Detail |
| PUT | `/api/complaints/{id}/status` | Admin | Update status |
| GET | `/api/complaints/my-complaints` | Student | Own complaints |

## 4. AI via Chat

Students can also submit complaints through the AI chat:

```
Student: "عندي شكوى على الدكتور — مش بيرد على الأسئلة"
    │
Intent: complaint_submit
    │
ComplaintModule:
  - Extracts: message, targetType (Doctor → "Doctor")
  - Injects from academic_context: userId, subjectOfferingId
  - POST /api/complaints
    Body: { message, targetType: "Doctor", userId, subjectOfferingId }
    │
Returns confirmation to student
```

## 5. AI Intelligence Reports (Hangfire)

`ComplaintIntelligenceJob` runs on three schedules:

| Schedule | Job | Output |
|----------|-----|--------|
| Daily | `GenerateDailyReportAsync()` | Yesterday's complaints, common themes |
| Weekly | `GenerateWeeklyReportAsync()` | Week's trends, top issue categories |
| Monthly | `GenerateMonthlyReportAsync()` | Monthly analysis, priority breakdown |

Reports are generated by calling FastAPI `/api/complaint_intelligence` which:
1. Fetches complaints from the specified period
2. Clusters by theme using LLM
3. Identifies top concerns and priority distribution
4. Returns structured summary for admin dashboard

---

## 6. Complaint Priority Escalation

| Priority | Trigger | Admin Notification |
|----------|---------|-------------------|
| Normal | Default | Included in daily digest |
| High | Student escalates | Immediate notification |
| Critical | Flagged by AI or admin | Urgent notification + escalation flag |

---

## 7. Status Lifecycle

```
Submitted → Pending
    │
Admin reviews → UnderReview
    │
    ├─ Issue resolved → Resolved
    └─ No action needed → Dismissed
```
