# Exams System — Complete Guide

## Overview

The exams system supports the full exam lifecycle from creation to grading. Doctors create exams manually, via AI generation, or by uploading a PDF. Students submit answers online. Auto-grading handles MCQ/TrueFalse instantly. Each student in a randomized exam receives a unique personal subset of questions drawn from a shared pool.

---

## Exam Types

| Type | Description |
|------|-------------|
| `Quiz` | Short quiz, few questions |
| `Midterm` | Mid-semester exam |
| `Final` | End-of-semester final |

## Exam Modes

| Mode | Description |
|------|-------------|
| `Structured` | Questions defined in the system (MCQ / TrueFalse / Essay) |
| `AI` | Questions generated by Claude AI |
| `File` | PDF uploaded by doctor — no in-system questions |

## Exam Status

| Status | Student Sees |
|--------|-------------|
| `Draft` | No |
| `Published` | Yes — can submit |
| `Closed` | Yes — read-only |

## Entities

### Exam

```csharp
public class Exam : BaseEntity
{
    public string     Title              { get; set; }
    public ExamType   Type               { get; set; }
    public int        TotalMarks         { get; set; }
    public DateTime   StartTime          { get; set; }
    public DateTime   EndTime            { get; set; }
    public ExamMode   Mode               { get; set; }
    public ExamStatus Status             { get; set; }
    public string?    FilePath           { get; set; }
    public Ulid       CreatedByDoctorId  { get; set; }
    public Ulid       SubjectOfferingId  { get; set; }

    // Randomized exam support
    public bool IsRandomized        { get; set; }  // Each student gets a unique subset
    public int  QuestionsPerStudent { get; set; }  // How many from the pool (0 = all)
}
```

### ExamQuestion

```csharp
public class ExamQuestion : BaseEntity
{
    public Ulid         ExamId        { get; set; }
    public string       QuestionText  { get; set; }
    public QuestionType QuestionType  { get; set; }  // MCQ=0, TrueFalse=1, Essay=2
    public string?      OptionsJson   { get; set; }  // JSON array of strings, MCQ only
    public string       CorrectAnswer { get; set; }  // Never sent to student role
    public int          Mark          { get; set; }
}
```

### StudentExamVariant

Created lazily when a student first opens a randomized exam. Stores the specific subset and display order of questions assigned to that student. Once created, it never changes — the student always sees the same questions.

```csharp
public class StudentExamVariant : BaseEntity
{
    public Ulid   ExamId          { get; set; }
    public Ulid   StudentId       { get; set; }
    public string QuestionIdsJson { get; set; }  // JSON array of ULID strings
}
```

Unique index on `(ExamId, StudentId)` — one variant per student per exam.

### ExamSubmission

```csharp
public class ExamSubmission : BaseEntity
{
    public Ulid      ExamId      { get; set; }
    public Ulid      StudentId   { get; set; }
    public DateTime  SubmittedAt { get; set; }
    public string    AnswersJson { get; set; }  // JSON array of { questionId, answerText }
    public double?   Score       { get; set; }
    public bool      IsGraded    { get; set; }
}
```

## API Reference

### Doctor Endpoints

#### `POST /api/exams?subjectOfferingId={id}`
Create a manual exam with questions.

```json
{
  "title": "Algorithms Midterm",
  "type": 1,
  "startTime": "2026-06-01T09:00:00Z",
  "endTime": "2026-06-01T11:00:00Z",
  "status": 0,
  "questions": [
    {
      "questionText": "What is the time complexity of merge sort?",
      "questionType": 0,
      "options": ["O(n)", "O(n log n)", "O(n²)", "O(log n)"],
      "correctAnswer": "O(n log n)",
      "mark": 10
    }
  ]
}
```

#### `POST /api/exams/generate-ai`
Ask Claude AI to generate an exam. Supports randomized mode.

```json
{
  "subjectOfferingId": "01JFKX...",
  "difficulty": "Medium",
  "questionCount": 30,
  "examType": "Final",
  "topics": ["Recursion", "Sorting"],
  "isRandomized": true,
  "questionsPerStudent": 10
}
```

- `questionCount` = pool size (total questions generated by AI)
- `questionsPerStudent` = how many each student sees (must be < `questionCount`)
- If `isRandomized: false` or `questionsPerStudent: 0` → all students get all questions
- Exam is created as `Draft` — doctor reviews and publishes manually

#### `POST /api/exams/upload-file?subjectOfferingId={id}`
Upload a PDF exam. No in-system questions; file is stored in R2.

#### `GET /api/exams/my-exams`
All exams across all offerings the doctor owns.

#### `GET /api/exams/offering/{offeringId}`
All exams for a specific offering.

#### `GET /api/exams/{id}/submissions`
All student submissions for an exam (includes score, isGraded, answersJson).

#### `POST /api/exams/{id}/auto-grade`
Grade all ungraded MCQ/TrueFalse submissions via exact string match on `CorrectAnswer`.
Returns `{ gradedCount: N }`.

#### `POST /api/exams/{id}/grade-submission`
Manually set a score for one submission (used for Essay questions).

```json
{ "submissionId": "01JFKX...", "score": 85.0 }
```

#### `PUT /api/exams/{id}` *(Admin)*
Update exam metadata (title, type, times, status).

#### `DELETE /api/exams/{id}` *(Admin)*
Soft-delete exam. Blocked if submissions exist.

### Student Endpoints

#### `GET /api/exams/{id}`
Returns the exam. For a non-randomized exam, returns all questions. For a randomized exam, use `my-variant` instead.
`correctAnswer` is always `null` for students.

#### `GET /api/exams/{id}/my-variant` ⭐ (Randomized Exams)
Returns the student's personal question subset for a randomized exam. The variant is created on the first call and locked — subsequent calls return the same questions in the same order.

```json
{
  "id": "01JFKX...",
  "isRandomized": true,
  "questionsPerStudent": 10,
  "questions": [
    {
      "id": "01JFKX...",
      "questionText": "What is the time complexity of merge sort?",
      "questionType": "MCQ",
      "options": ["O(n)", "O(n log n)", "O(n²)", "O(log n)"],
      "mark": 10,
      "correctAnswer": null
    }
  ]
}
```

#### `GET /api/exams/student-exams`
All published exams across the student's enrolled offerings.

#### `GET /api/exams/{id}/by-code?code={code}`
Look up an exam by its short code.

#### `POST /api/exams/{id}/submit`
Submit answers. Only send answers for questions in the student's variant.

```json
{
  "examId": "01JFKX...",
  "answers": [
    { "questionId": "01JFKX...", "answerText": "O(n log n)" },
    { "questionId": "01JFKX...", "answerText": "True" }
  ]
}
```

Response `201`:
```json
{ "submissionId": "01JFKX...", "message": "Exam submitted successfully." }
```

#### `GET /api/exams/{id}/my-submission`
The student's own submission and score for an exam.

## Randomized Exam Flow

```
Doctor:
  POST /api/exams/generate-ai  { questionCount: 30, questionsPerStudent: 10, isRandomized: true }
  → Claude generates 30 questions (the pool)
  → Exam saved with IsRandomized=true, QuestionsPerStudent=10
  → Doctor reviews draft, publishes

Student A opens exam:
  GET /api/exams/{id}/my-variant
  → No StudentExamVariant exists yet
  → System shuffles 30 question IDs, picks first 10
  → Creates StudentExamVariant { ExamId, StudentId, QuestionIdsJson }
  → Returns those 10 questions (no correctAnswer)

Student B opens exam:
  GET /api/exams/{id}/my-variant
  → Different shuffle → different 10 questions
  → Student B never sees Student A's exact set

Student A opens exam again:
  GET /api/exams/{id}/my-variant
  → Existing variant found → same 10 questions returned (locked)

After exam closes:
  Doctor calls POST /api/exams/{id}/auto-grade
  → For each submission, grades against only that student's variant questions
```

## Security Rules

| Rule | Implementation |
|------|---------------|
| `correctAnswer` never sent to students | `MapToDto(includeCorrectAnswers: false)` for Student role |
| Doctor can only manage own offering's exams | `offering.DoctorId != doctorId` check in service |
| Students can only see Published exams | `status == Published` filter in `GetStudentEnrolledExamsAsync` |
| One submission per student per exam | Unique index on `(ExamId, StudentId)` in ExamSubmissions |
| Variant locked after first open | `StudentExamVariant` created once, never modified |
| Cannot delete exam with submissions | Guard in `DeleteExamAsync` |

## Automated Jobs

### ExamReminderJob
Runs every 30 minutes. Sends push notifications to enrolled students:
- ~24 hours before exam start
- ~2 hours before exam start

Students get exactly 2 reminders per exam. Duplicate notifications are suppressed.

## Question Types Reference

| `questionType` | `options` | Expected `answerText` | Auto-Gradable |
|---|---|---|---|
| `MCQ` (0) | Array of 4 strings | One of the option strings exactly | Yes |
| `TrueFalse` (1) | null | `"True"` or `"False"` | Yes |
| `Essay` (2) | null | Free text | No — manual grading |
